Positioning

定位相关

它是不是一个普通聊天系统

不是。它是一个让 AI 在规则内调用和执行真实能力的平台，当前最成熟的执行资产是 Python 脚本；同时 Web 端上传注册已经支持 Python / Shell / Bat(Cmd) / Node.js。

它适合哪些人

适合已经有脚本、自动化任务、数据处理需求，或需要把这些能力变成统一入口、统一交付和统一治理的人。

免费版是不是企业后台裁剪版

不是。免费版面向个人探索者，用来快速体验能力接入、AI 调度与执行反馈；团队与企业场景再进入更完整的协作和治理路径。

企业线和免费线是不是两套完全分裂的系统

不是。它们共享同一套执行与安全底座，只是面向的使用规模、协作深度和交付方式不同。

现在是不是只有一条正式路径

不是。当前公开口径已经拆成四层：免费线、本地脚本位增购、标准团队版正式租户入口，以及企业版 / 私有化交付。

Safety

安全、隔离与权限边界

AI 会不会越权执行

当前平台强调 AI 在规则内工作。推荐范围、执行边界、人工确认和审计留痕都不是可有可无的装饰层。

一个客户能不能看到另一个客户的能力

不能。无论是共享 SaaS 还是企业单租户交付，客户默认都只能看到自己租户内被授权的能力。

平台如何保证不同客户之间隔离

平台默认按客户边界隔离能力、数据和权限，确保不同客户之间互不可见、互不干扰。

AI 会不会跨租户推荐别的客户能力

不会。当前推荐范围受租户与权限共同约束，不是“模型想到什么就推什么”。

对话执行的大模型地址和 Token 是统一一套吗

不是只能统一一套。平台可以维护默认配置，正式租户也可以维护自己的本地配置；如果租户本地配置不完整，运行时会继续回退平台默认配置。现在公开口径还要再补一层：平台超管维护平台模型库，租户管理员维护本租户的大模型列表；超管平台模型要先授权到租户，再下发到具体用户；用户在公告中心确认后，模型才会进入自己的对话执行可选范围。个人免费版普通用户不需要单独的大模型列表页，也可以直接在对话执行里切换自己已授权的模型。为了避免敏感信息横向暴露，完整 Token 只对配置方可见，另一侧只显示遮罩。

为什么代码里会看到 intelligent_brain，但页面写的是对话执行

intelligent_brain 是当前代码、目录和接口层继续保留的兼容技术标识，现阶段不做整库重命名；页面、菜单、官网和交付资料统一显示为“对话执行”。两者指向的是同一套能力域。

Delivery

交付与热更新

是不是一定要私有化才能用

不一定。标准团队协作当前可以先走共享 SaaS 正式租户入口；只有涉及更强隔离、内网、本地桥接或正式项目验收时，才更适合继续走企业交付或私有化。

私有化部署测试页面在哪里看

当前统一口径是：私有化部署测试页面可在 private-test.execfabric.cn 查看。

热更新是不是要重部署整个平台

不是。客户脚本通过审核、同步、注册和授权后，可以在当前租户内生效，不需要整个平台重新部署。

客户能不能自己上传脚本后直接正式上线

当前 1.0 默认不这样做。平台仍保留审核、注册、授权和权限验证流程，不支持绕过平台直接正式上线。

客户能不能自己上传任意前端页面直接上线

不能。当前菜单可挂已有页面、外链和 iframe，但新增正式前端页面仍需要开发、打包和部署；共享 SaaS 不支持租户自行上传任意 Vue / JS 成为正式页面。

热更新会不会影响其他客户

不会。脚本更新后只在所属租户范围内生效，其他租户不会自动获得可见或可执行权限。

是不是所有项目都默认包含文件上传和结果下载

不一定。如果项目需要处理输入文件和结果产物，平台可以提供文件上传、批次状态和结果下载链路；是否开放以具体项目范围为准。

上传文件后，脚本怎么知道该处理哪一个文件

不是让脚本自己去猜。当前对话执行底部快捷区的“上传文件”按钮会把这次上传绑定成一个 uploadBatchId，确认执行时后端再把这批文件的快照传给脚本。Python / Shell / Bat(Cmd) / Node.js 脚本都可以直接读取 EXECFABRIC_UPLOAD_BATCH_NO、EXECFABRIC_UPLOAD_INPUT_FILES_JSON、EXECFABRIC_UPLOAD_FILE_SERVICE_JSON 和 EXECFABRIC_SKILL_INPUT_PAYLOAD_JSON，所以处理的是当前绑定批次，而不是历史目录里“最新那个文件”。

正式交付后客户通常会拿到哪些说明

通常至少包括访问入口、账号方式、客户使用说明、客户脚本热更新说明；如果项目包含公告、客户须知或结果下载入口，也应一起交待清楚。

CLI & Local

CLI、本地脚本和 Agent

CLI 现在是不是概念功能

不是。execfabric-cli 已支持 login、register、list、run、agent describe、agent bind 和 agent start，适合本地接入、调试和前期验证。

本地 Agent 是不是已经正式商用

还不是。本地 Agent 相关能力仍在持续完善，目前可以先按本地桥接与执行骨架理解。

本地脚本和内网资源能不能接入平台

可以，但通常不是让浏览器直接碰本地路径，而是通过 CLI / Agent 路线承接本地执行桥接。

是不是所有脚本类型都已经完整支持远程入库

网页端上传注册现在已经支持 Python / Shell / Bat(Cmd) / Node.js 文本脚本；页面会按当前云端环境自动判断是否开放 bat / cmd，非 Windows 云端会直接限制。CLI 远程注册当前也已支持 Python / Shell / Bat(Cmd) / Node.js；如果是本机路径登记，CLI / Local Agent 已可登记 Python / Shell / Bat / Node。需要注意的是，bat / cmd 在线执行依赖 Windows 运行时。

Stage

当前阶段与升级路径

现在已经做出什么

企业 1.0 交付主线、个人免费入口、CLI 最小桥梁能力和外部文档都已可用，可以支撑公开讲解、免费线体验和部分真实场景。

标准化小团队方案是不是已经完全开放

还没有完全开放成最终完成态，但璇络标准 SaaS 共享租户前端已经建立，月租 / 年租升级路径也已经存在，后续继续收口的是品牌化、初始化和更细菜单边界。

体验页适合帮我判断什么

它适合先感受执行链路、理解平台定位，并判断自己更适合继续走免费线、本地脚本位、标准团队版还是企业咨询。当前体验页左侧是 Python `task_brief_demo.py`，右侧是 Shell `clean_log_demo.sh`，对应脚本来自正式的免费线体验脚本目录。

注册成功后会去哪里

当前注册成功后会先进入独立结果页，再回到登录与个人主链路，不再只是靠提示框承接。

execfabric-hack 是不是客户正式入口

不是。它主要用于本地多入口聚合、联调和同步，不作为客户长期正式运行入口；客户正式入口还是个人免费端、共享 SaaS 正式租户或企业交付前端。
