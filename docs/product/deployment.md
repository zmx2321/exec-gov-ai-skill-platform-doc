  <section class="brand-grid brand-grid--three">
    <article class="brand-card">
      <p class="brand-kicker">01</p>
      <h2>单租户交付模式</h2>
      <ul class="brand-list">
        <li>适合企业客户项目交付</li>
        <li>需要独立访问地址</li>
        <li>需要和其他客户完全隔离</li>
        <li>一个客户一个租户，能力与数据边界清晰</li>
      </ul>
    </article>
    <article class="brand-card">
      <p class="brand-kicker">02</p>
      <h2>平台治理模式</h2>
      <ul class="brand-list">
        <li>平台方统一管理多个客户</li>
        <li>需要公告、模板、日志和台账总控</li>
        <li>需要跨租户治理但不打破隔离</li>
        <li><code>public</code> 作为平台治理域</li>
      </ul>
    </article>
    <article class="brand-card">
      <p class="brand-kicker">03</p>
      <h2>免费入口模式</h2>
      <ul class="brand-list">
        <li>适合产品体验入口和个人开发者试用</li>
        <li>更强调上手速度和体验路径</li>
        <li>为后续团队版、企业版升级提供前置入口</li>
      </ul>
    </article>
  </section>

  <section class="brand-grid brand-grid--two">
    <article class="brand-card">
      <p class="brand-kicker">04</p>
      <h2>本地混合执行模式</h2>
      <ul class="brand-list">
        <li>适合文件和数据不能全部上云</li>
        <li>任务依赖本地环境或内网资源</li>
        <li>平台负责入口、规划、权限和审计</li>
        <li>本地环境负责执行，CLI / Agent 负责桥接</li>
      </ul>
    </article>
    <article class="brand-card">
      <p class="brand-kicker">05</p>
      <h2>未来边缘与设备模式</h2>
      <ul class="brand-list">
        <li>这是 <code>4.0</code> 的公开方向之一</li>
        <li>能力单元不再局限于脚本</li>
        <li>执行端可能是本地 Agent、边缘节点或设备插件</li>
        <li>Web 控制面继续作为统一治理入口</li>
      </ul>
    </article>
  </section>
